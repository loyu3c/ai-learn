01Claude 是什麼？

一個會「對話」的 AI 助理

Claude 是由美國 AI 公司 Anthropic 打造的大型語言模型（LLM）。 你可以把它想像成一位反應很快、讀過大量資料、可以用文字（也能看圖片、讀文件） 跟你聊天、幫你寫作、寫程式、整理資料的助理。

你可能已經聽過的名字

Claude Code（你現在使用的這個工具背後的技術）、 claude.ai（網頁版聊天介面）、 Claude API（讓開發者把 Claude 整合進自己產品的介面）， 其實都是同一個 Claude 模型，只是「使用的入口」不同。

「Anthropic」跟「Claude」的關係

Anthropic 是公司名稱（研發單位），Claude 是產品 / 模型名稱 （就像 Google 是公司、Gemini 是它的模型；OpenAI 是公司、GPT 是它的模型）。

02模型家族比較

Claude 不是只有一個模型，而是一個「家族」。不同型號在 聰明程度、回覆速度、使用成本之間做了不同的取捨， 你可以依照任務難度挑選合適的模型。

最強

Opus 4.8

推理能力最強、思考最深入，適合最困難、最複雜的任務。

  • 複雜的程式架構設計
  • 長篇研究與深度分析
  • 需要多步驟推理的任務
速度：中等　｜　成本：較高
均衡首選

Sonnet 5

智慧與速度的最佳平衡點，是目前最常用的日常主力模型。

  • 日常對話、寫作、翻譯
  • 程式撰寫與除錯
  • 文件摘要、資料整理
速度：快　｜　成本：中等
最快

Haiku 4.5

體積最小、回覆最快、成本最低，適合大量、即時、簡單的任務。

  • 即時客服對話
  • 簡單分類、資料抽取
  • 需要低延遲的應用
速度：最快　｜　成本：最低
創意向

Fable 5

偏向創意寫作與敘事表現調校的模型，適合說故事、創作類任務。

  • 故事、劇本、文案創作
  • 角色扮演式對話
  • 需要豐富文筆的內容
速度：快　｜　成本：中等

💡 小提醒：不確定要選哪一個時，先用 Sonnet 系列就對了—— 它是多數人日常使用的預設選擇，簡單任務用小模型（Haiku）省錢省時間， 困難任務再升級到大模型（Opus）。

03核心功能導覽

點選下方分類，看看 Claude 各項基礎功能的中文說明與使用情境。

💬 對話與寫作（Chat）

是什麼：最基本的功能。用自然語言打字跟 Claude 對話，它會理解你的問題並用文字回覆。

可以做什麼：回答問題、腦力激盪、寫文章／email／社群貼文、潤稿、翻譯、角色扮演練習對話等。

範例提示詞

「幫我把這封 email 改得更正式、更有禮貌，但不要太長：〔貼上你的內容〕」

👨‍💻 程式撰寫與除錯（Coding）

是什麼：Claude 能讀懂、撰寫、解釋多種程式語言的程式碼，也能幫你找出並修正錯誤（bug）。

可以做什麼：從零寫一個小工具、解釋一段看不懂的程式碼、重構程式、寫測試、把錯誤訊息貼給它幫你抓蟲。 你現在使用的 Claude Code（命令列工具）就是把這項能力包裝成可以直接操作專案檔案的開發助理。

範例提示詞

「這段 Python 程式跑出 IndexError，幫我找出原因並修好：〔貼上程式碼與錯誤訊息〕」

🖼️ 圖片與文件理解（Vision / 多模態）

是什麼：Claude 除了讀文字，也能「看」圖片、截圖、圖表，以及讀取 PDF、Word 等文件內容。

可以做什麼：看一張圖表幫你解讀數據、看網站截圖抓 UI 問題、把手寫筆記轉成文字、幫你摘要一份 PDF 報告。

範例情境

上傳一張數學題目的照片，請它「一步一步解釋解題過程」。

📚 長文本閱讀（Long Context）

是什麼：Claude 一次可以「讀」非常長的內容（可達數十萬字），不會像短記憶的助理一樣忘記前面說過什麼。

可以做什麼：一次丟一整本書、一整份法律合約、一整個程式碼庫，請它做摘要、找重點、比對前後矛盾之處。

範例提示詞

「以下是一份 50 頁的合約，幫我整理出對甲方（我方）不利的條款，並用條列式列出：〔貼上全文〕」

🛠️ 工具使用（Tool Use / Function Calling）

是什麼：讓 Claude 不只是「回答文字」，還能在需要時去「呼叫」外部工具或程式， 例如查天氣 API、搜尋網路、計算機、查詢資料庫，取得即時或精確的結果後再回覆你。

相關名詞：MCP（Model Context Protocol） 是 Anthropic 推出的一套標準，讓 Claude 可以用統一的方式連接各種外部工具與資料來源 （例如 Google Drive、Slack、GitHub），大幅擴充它能完成的任務範圍。

生活化比喻

Claude 本身像一個很聰明的大腦，工具使用／MCP 就像是幫這顆大腦裝上手腳， 讓它能真的去「操作」外部的東西，而不只是動腦想。

🧩 Artifacts（即時預覽產出）

是什麼：在 claude.ai 網頁版對話時，如果請 Claude 產生程式碼、網頁、圖表、文件等「較完整的內容」， 它會在對話旁邊開一個獨立的預覽視窗即時顯示成品，而不是全部混在聊天紀錄裡。

可以做什麼：直接在瀏覽器裡看到你要的網頁長怎樣、圖表畫出來的樣子、程式碼的執行結果， 方便反覆修改、迭代。

📁 Projects（專案功能）

是什麼：可以建立一個「專案」，把相關的文件、資料、自訂指示（例如公司的寫作風格、 常用術語）都放在裡面，之後在這個專案內的每次對話，Claude 都會自動參考這些背景資料。

可以做什麼：幫團隊建立一個「客服知識庫專案」、「產品文件專案」，讓每次提問都基於同一批資料回答， 不用每次重新貼資料。

🖥️ Computer Use（電腦操作）

是什麼：讓 Claude 直接「看」你的電腦螢幕畫面，並模擬滑鼠點擊、鍵盤輸入， 像真人一樣操作電腦上的軟體，來完成多步驟的自動化任務。

可以做什麼：自動化重複性的表單填寫、跨多個網站蒐集資訊、測試軟體介面流程等， 目前多用於開發者打造自動化代理人（Agent）應用。

04如何開始使用

1

網頁版 claude.ai

最簡單的入門方式。註冊帳號後即可在瀏覽器裡直接跟 Claude 對話， 適合日常聊天、寫作、學習使用，不需要任何技術背景。

2

Claude API

給開發者使用的介面。透過程式呼叫 Claude，把它整合進自己開發的 App、網站或內部工具中，依實際用量計費。

3

Claude Code（CLI）

在終端機（命令列）中直接使用的開發助理，可以讀寫你專案裡的檔案、 執行指令、幫你完成程式開發任務——就是你現在正在體驗的工具。

05提示技巧（Prompting 入門）

問得好，答案才會好。以下是幾個新手最有感的技巧：

1. 給清楚的角色與目標

❌「幫我寫一段文案」

✅「你是一位社群小編，幫我寫一則 IG 貼文文案，推廣新上市的手沖咖啡豆，語氣要輕鬆活潑」

2. 提供背景資料

把需要的原始資料、範例、限制條件直接貼給 Claude，而不是只描述，減少它「猜」的空間。

3. 說明期望的格式

指定「用條列式」「用表格」「限制在 100 字以內」「用繁體中文回答」，輸出會更符合你的需求。

4. 拆解複雜任務

困難的任務可以先請它「列出步驟」，確認方向沒問題後，再逐步請它完成每一步。

5. 允許它說「不知道」

可以在提示詞加上「如果不確定，請直接說不確定，不要編造答案」，降低 AI 幻覺（瞎掰）的風險。

6. 善用追問與修改

第一次的回答不滿意很正常，直接接著說「太長了，幫我濃縮成 3 點」即可，不需要重新問一次。
